Three-Tier Integration

Terra offers three progressively detailed integration tiers, from zero-code setup to full manual control.

Tier 1

Zero-Code Auto

A single Terra.start() call instruments CoreML and HTTP AI requests automatically. No application code changes.

Tier 2

@Traced Macro

Annotate async functions with @Traced(model:). Auto-detects prompt and maxTokens parameters at compile time.

Tier 3

Closure Wrappers

Full control over span lifecycle, attributes, events, and error recording via typed Scope<Kind> wrappers.

Swift — Tier 1
import Terra

try Terra.installOpenTelemetry(.init(
  persistence: .init(storageURL: Terra.defaultPersistenceStorageURL())
))
Terra.install(.init(privacy: .default))
try Terra.start()
Swift — Tier 2
@Traced(model: "llama-3.2-1B")
func generate(prompt: String, maxTokens: Int = 512) async throws -> String {
    // your inference code
}
Swift — Tier 3
let result = try await Terra.withInferenceSpan(
    .init(model: "llama-3.2-1B", prompt: prompt, maxOutputTokens: 256)
) { scope in
    let output = try await myModel.generate(prompt)
    scope.setAttributes(["custom.key": .string("value")])
    return output
}

04

Auto-Instrumentation: CoreML

TerraCoreML

Transparent instrumentation of all CoreML predictions via Objective-C runtime method swizzling.

Captured per prediction

  • Timing: Monotonic duration via ContinuousClock (nanosecond precision)
  • Compute units: all, cpu_only, cpu_and_gpu, cpu_and_ane
  • Model name: Resolved from metadata, display name, or fallback
  • Memory delta: Resident memory change during prediction
  • GPU metrics: Compute time via TerraMetalProfiler
Smart dedup: Checks for an active span before creating a new one to avoid double-tracing in nested contexts.
Swift
CoreMLInstrumentation.install(.init(
    excludedModelNames: ["fast-classifier"]
))
05

Auto-Instrumentation: HTTP AI APIs

TerraHTTPInstrument

Callback-based URLSession instrumentation. Recognizes OpenAI, Anthropic, Google, Together.ai, Mistral, Groq, Cohere, Fireworks.ai, Ollama (11434), LM Studio (1234).

From streaming responses (SSE / NDJSON)

  • Time to first token (TTFT), tokens/sec, chunk count
  • Per-stage durations (prompt eval, decode, model load)
  • Stall detection (gap ≥ 300ms)
  • Inter-token gap tracking
Safety: Max 10 MiB body parsing. No prompt/completion text captured — only structured metadata.

Foundation Models

TerraFoundationModels

Wraps Apple's LanguageModelSession (macOS 26+) with traced equivalents for text, structured output, and streaming.

Swift
let session = TerraTracedSession(model: .default)

let text = try await session.respond(to: "Hello")

let result = try await session.respond(
    to: "Classify this",
    generating: Classification.self
)

for try await chunk in session.streamResponse(to: "Tell me a story") {
    print(chunk)
}

Privacy & Content Control

No content is captured by default.

PolicyBehavior
.never (default)Never capture prompt/completion/thinking text
.optInCapture only with explicit CaptureIntent.optIn
.alwaysAlways capture content
RedactionBehavior
.dropSilently remove excluded content
.lengthOnlyReplace with character count
.hashSHA256 (default)Anonymized SHA-256 hash

Anonymization

  • HMAC-SHA256 with rotating keys (24h default)
  • Per-purpose secrets prevent cross-correlation
  • Key ID attached to spans for audit trail

Compliance & Export Controls

Swift
let compliance = CompliancePolicy(
    exportControl: .init(
        allowedRuntimes: [.coreML, .foundationModels],
        blockOnViolation: true
    )
)
  • Retention: Max age (7 days), max storage (256 MB), LRU/oldest-first eviction
  • Audit logging: Policy violations, schema rejections, version mismatches (max 1,024 buffered)
  • Cross-process consent boundary prevents telemetry leaking

OpenTelemetry Integration

Built on OpenTelemetry Swift. OTLP/HTTP export, Instruments.app signposts, automatic sessions.

Swift
try Terra.installOpenTelemetry(.init(
    tracesEndpoint: URL(string: "http://localhost:4318/v1/traces")!,
    enableSignposts: true,
    enableSessions: true,
    samplingRatio: 1.0
))

Span processors

  • TerraSpanEnrichmentProcessor — adds privacy, schema version, redaction, anonymization key
  • TerraSessionSpanProcessor — attaches session ID to Terra spans
